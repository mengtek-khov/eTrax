# Project Agent Rules

- Write all implementation logic and work notes to DEVELOPMENT.LOG in the project root.
- If any instruction is unclear or conflicts with another instruction, ask the user before proceeding.
- Keep coding style consistent with the existing codebase.
- Suggest better approaches when they provide clear benefits.
- Implement Python code as reusable modules with clean interfaces so it can integrate with other frameworks.
- Runtime priority: keep the project standalone-first on Windows now.
- Integration priority for future adapters: Frappe first, then other frameworks.
- Product domain priority: Telegram bot builder with modular, composable workflow blocks.
